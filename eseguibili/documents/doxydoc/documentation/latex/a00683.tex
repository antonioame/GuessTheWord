\doxysubsection{Descrizione dettagliata}
Classe base astratta per tutti i payload scambiati in rete tra server e client. 


\begin{DoxyItemize}
\item 
\end{DoxyItemize}

Stabilisce il contratto fondamentale del protocollo di rete del progetto. Ogni pacchetto serializzabile viaggiante sui socket deve estendere questa classe. Assicura l\textquotesingle{}autoconsapevolezza temporale (timestamp) e l\textquotesingle{}interoperabilità verso la GUI tramite la conversione in {\ttfamily \doxylink{a00647}{Callback\+DTO}}.


\begin{DoxyItemize}
\item 
\end{DoxyItemize}

{\bfseries{FLUSSO DI COMUNICAZIONE CLIENT-\/\+SERVER\+:}}


\begin{DoxyEnumerate}
\item {\bfseries{Autenticazione\+:}} Il client invia una {\ttfamily \doxylink{a00687}{Login\+Request}}. Il server controlla i dati nel DB e risponde con {\ttfamily \doxylink{a00691}{Login\+Response}}, specificando anche se l\textquotesingle{}utente ha privilegi di amministratore. 
\item {\bfseries{Registrazione\+:}} Se l\textquotesingle{}utente non ha un account, invia una {\ttfamily \doxylink{a00695}{Register\+Request}}. Il server inserisce i dati e risponde con {\ttfamily \doxylink{a00699}{Register\+Response}}.


\item {\bfseries{Richiesta Partita\+:}} L\textquotesingle{}utente loggato invia una {\ttfamily \doxylink{a00703}{Play\+Request}} per cercare un avversario. Il server risponde con {\ttfamily \doxylink{a00707}{Play\+Response}}, indicando se la partita inizia subito (MATCH\+\_\+\+FOUND) o se deve aspettare (WAITING). 
\item {\bfseries{Inizio Partita\+:}} Appena ci sono due giocatori, il server invia a entrambi un messaggio {\ttfamily \doxylink{a00711}{Game\+Start}} contenente la sfida cifrata, il timer e i dati dell\textquotesingle{}avversario. 
\item {\bfseries{Fase di Gioco\+:}} 
\begin{DoxyItemize}
\item Il giocatore elabora la risposta e la invia con {\ttfamily \doxylink{a00715}{Answer\+Submission}}. 
\item Se un giocatore risponde, il server avvisa l\textquotesingle{}altro in tempo reale tramite {\ttfamily \doxylink{a00723}{Opponent\+Answered}}. 
\end{DoxyItemize}
\item {\bfseries{Fine Partita\+:}} Il server elabora i tempi e decreta un vincitore, comunicandolo ai due client tramite il messaggio {\ttfamily \doxylink{a00719}{Game\+Result}}.


\item {\bfseries{Storico\+:}} Dalla lobby, il client può richiedere lo storico delle proprie partite con {\ttfamily \doxylink{a00739}{History\+Request}}. Il server interroga il DB e risponde con {\ttfamily \doxylink{a00743}{History\+Response}}.


\item {\bfseries{Disconnessione\+:}} 
\begin{DoxyItemize}
\item Se un client esce dal gioco regolarmente, invia {\ttfamily \doxylink{a00735}{Client\+Disconnect}}. 
\item Se un client cade o si disconnette durante un\textquotesingle{}attesa o una partita, il server avvisa l\textquotesingle{}avversario superstite inviando {\ttfamily \doxylink{a00731}{Opponent\+Disconnected}}. 
\end{DoxyItemize}
\end{DoxyEnumerate}
