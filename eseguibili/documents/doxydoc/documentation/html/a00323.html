Descrizione dettagliata

Implementazione lato client della connessione di rete.

Specializza NetworkConnection fornendo la logica per instaurare e gestire un singolo canale di comunicazione bidirezionale verso il server di gioco. Nasconde la complessità della gestione multi-canale del padre, fissando implicitamente tutte le comunicazioni sull'indice di canale 0.

Costruttore completo per inizializzare il modulo di rete del client.

Prepara i parametri di connessione e le callback, ma non avvia immediatamente la comunicazione. Per connettersi fisicamente è necessario invocare il metodo ereditato connect().

[in]serverPortLa porta del server.
[in]onReceiveCallback invocata dinamicamente ad ogni pacchetto ricevuto.
[in]onDisconnectCallback di emergenza/pulizia invocata in caso di caduta della connessione.

Socket gruppo05.gtwclient.networking.ClientConnection.createSocket ( ) throws IOException

Instanzia e avvia attivamente una connessione TCP verso il server.

Questo metodo viene invocato automaticamente in modo asincrono dal thread di setup di NetworkConnection#connect(). A differenza del server che attende passivamente, il client lancia un handshake di rete esplicito.

Restituisce
Una Socket validata, aperta e collegata al server.
Eccezioni
IOExceptionSe il server risulta offline, irraggiungibile, o la connessione viene rifiutata.

int gruppo05.gtwclient.networking.ClientConnection.expectedChannels ( )

Indica il numero di canali di rete che questa classe deve inizializzare.

Poiché un client comunica esclusivamente in un rapporto 1-a-1 con il server di gioco (e mai P2P con gli altri client), il valore atteso è costantemente 1.

Restituisce
Il valore intero fisso 1.
